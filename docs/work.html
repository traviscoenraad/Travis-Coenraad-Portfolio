Security Log Analyser

Technologies Used: Python, argparse, dataclasses, datetime, unittest

A command-line tool that parses authentication log files and automatically detects security threats including brute force attacks, persistent suspicious IPs and abnormal login frequency using rule-based detection with sliding window algorithms. Built with a modular architecture separating ingestion, parsing, detection and reporting concerns. This project demonstrates practical cybersecurity knowledge and professional software engineering practices including Git feature branching and unit testing.

What I'd improve: I would add a web interface allowing users to upload log files through a browser rather than the command line which makes the tool accessible to non-technical security staff.

SecuritySuite

Technologies Used: Java, Data Structures (Stack, HashMap), File I/O, Swing (JOptionPane)

A Java-based file system security tool that allows users to search for files using a depth-first search algorithm and analyse directory contents including file counts, total sizes and extension breakdowns. The tool uses real security concepts including path traversal attack prevention, read permission validation, hidden file detection and timestamped audit logging which demonstrates both data structures knowledge and practical security awareness.

What I'd improve: I would replace the JOptionPane GUI with a proper Java Swing interface or web frontend to make the tool more user friendly and add unit tests using JUnit to formally verify each security method behaves correctly.

SwiftChat Management System — Systems Analysis & Design

Technologies Used: UML (Use Case & Class Diagrams), Entity Relationship Diagrams, SQL, Figma, Microsoft Project (Gantt Chart), Microsoft Teams

As part of a university group project, we produced a full systems analysis and design proposal for SwiftChat, a cloud-based messaging platform struggling with entirely manual operations. Working in a team of five, we delivered a professional systems proposal covering feasibility analysis, risk assessment, stakeholder interviews, requirements gathering, UML diagrams, database design and three UI prototypes — proposing a fully automated replacement system within a £60,000 budget and five month timeframe.

My Role: As group leader and project manager I organised weekly Microsoft Teams meetings, planned and delegated tasks week by week, reviewed all work before it entered the final document and kept the project on track when three members reduced their contribution. I personally completed the feasiblity analysis, requirements gathering and structural analysis to maintain quality and meet deadlines.
